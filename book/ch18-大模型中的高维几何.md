# 第十八章：大模型中的高维几何

> "国王"减去"男人"加上"女人"等于"王后"——这不是文字游戏，这是向量空间里的算术。

---

## 一个惊人的等式

2013 年，Google 的 Tomas Mikolov 和他的同事发表了一篇论文，标题不起眼，内容却震动了整个自然语言处理领域。

他们做了这样一件事：把英语中的每个单词映射成一个几百维的向量——一串几百个浮点数。映射的方式不是人为指定的，而是通过一个叫 **Word2Vec** 的模型从大量文本中自动学出来的。基本思路是：如果两个词经常出现在相似的上下文中（比如"猫"和"狗"经常出现在"宠物""喂食""可爱"等词的附近），那它们的向量就应该相近。

这个思路并不新鲜——"通过上下文理解词义"是语言学的老生常谈。新鲜的是他们发现的那些向量具有的性质。

他们把训练好的词向量做了一系列"向量算术"——两个向量相减、相加——然后查找结果向量最近的邻居，看看对应什么词。结果令人目瞪口呆：

$$\vec{\text{king}} - \vec{\text{man}} + \vec{\text{woman}} \approx \vec{\text{queen}}$$

"国王"的向量减去"男人"的向量加上"女人"的向量，得到的结果向量，在整个词汇表中最接近"王后"的向量。

不止这一个。$\vec{\text{Paris}} - \vec{\text{France}} + \vec{\text{Italy}} \approx \vec{\text{Rome}}$——巴黎之于法国，如同罗马之于意大利。$\vec{\text{bigger}} - \vec{\text{big}} + \vec{\text{small}} \approx \vec{\text{smaller}}$——从"大"到"更大"的变化方向，和从"小"到"更小"的变化方向，竟然近似平行。

这些等式在说什么？

它们在说：**语义关系在向量空间中表现为方向**。"男人"到"女人"的语义关系（性别转换），在向量空间里对应一个大致固定的方向——从 $\vec{\text{man}}$ 指向 $\vec{\text{woman}}$ 的方向，和从 $\vec{\text{king}}$ 指向 $\vec{\text{queen}}$ 的方向，近似平行。"法国"到"巴黎"的语义关系（国家到首都），也对应一个大致固定的方向。

向量相减 $\vec{b} - \vec{a}$ 得到从 $a$ 到 $b$ 的"位移向量"——这是第五章就讲过的最基本的向量运算。而 Word2Vec 的发现告诉我们：在训练出来的词向量空间里，某些语义关系对应着近似一致的位移向量。

这是线性代数与语义学的一次令人惊叹的碰撞。

当然，必须加一个重要的限定：**并非所有语义关系都能如此稳定地被向量算术捕捉。** "king - man + woman = queen"是一个教科书级的漂亮例子，但如果你换成更微妙的语义关系——比如反义词、隐喻、文化特定的概念——向量算术的效果就参差不齐了。研究者后来发现，这种"线性类比"主要在某些规则性强、出现频率高的语义关系上表现良好。

但即使打了折扣，Word2Vec 的发现仍然是一个里程碑：它第一次大规模地展示了，**通过数据训练出来的高维向量空间，能够自发地涌现出近似方向结构来编码语义关系。** 这为后来整个"向量化一切"的范式奠定了基础。

## Embedding：从符号到向量

Word2Vec 的成功开启了一种全新的范式——**Embedding**（嵌入）。

在 Word2Vec 之前，计算机处理语言的方式本质上是符号化的。"猫"是一个符号，"狗"是另一个符号，它们之间的关系需要人为定义（比如在知识图谱里手动标注"猫和狗都是宠物"）。计算机不知道"猫"和"狗"之间有什么内在的相似性——在它眼里，"猫"和"狗"的差别跟"猫"和"引力波"的差别没有任何不同。

Embedding 的思路是：**给每个离散对象——一个词、一个用户、一件商品——分配一个连续的高维向量**，而且这个向量不是手工设计的，而是从数据中学出来的。

用线性代数的视角来看，Embedding 的过程惊人地简单。

假设你的词汇表有 $V$ 个词。每个词可以用一个 **one-hot 向量** 来表示——一个 $V$ 维向量，只有对应位置是 1，其余全是 0。"猫"可能是 $(0, 0, \ldots, 1, \ldots, 0, 0)$，"狗"是 $(0, 0, \ldots, 0, 1, \ldots, 0)$。

one-hot 向量彼此正交，距离相等。"猫"和"狗"之间的欧氏距离等于 $\sqrt{2}$，"猫"和"引力波"之间的距离也是 $\sqrt{2}$——没有任何结构，纯粹是符号的机械编码。

现在，定义一个 **Embedding 矩阵** $E$，大小为 $d \times V$（$d$ 是嵌入维度，通常远小于 $V$——比如 $V = 50000$ 个词，$d = 300$ 维）。把 one-hot 向量 $\mathbf{e}_i$（第 $i$ 个词）乘以这个矩阵：

$$\text{embedding}_i = E \mathbf{e}_i$$

矩阵乘以 one-hot 向量的结果，就是取出矩阵的第 $i$ 列。所以 Embedding 矩阵的每一列就是一个词的向量表示，而"查找一个词的 Embedding"这个操作，在数学上就是一次矩阵-向量乘法——只不过因为输入是 one-hot 的，所以这次乘法退化成了"取一列"。

Embedding 矩阵 $E$ 的 $d \times V$ 个数字，就是需要从数据中学习的参数。训练完成后，$E$ 的每一列就是对应词的"向量身份证"——而且不同词的"身份证"之间的几何关系（距离、方向、夹角），自动编码了语义信息。

这个想法很快就从文本扩展到了一切离散对象。推荐系统里，每个用户和每件商品都有一个 Embedding 向量——回忆第十六章的矩阵分解，$P$ 的每一行就是一个用户的 Embedding，$Q$ 的每一行就是一件商品的 Embedding。音乐推荐、广告投放、社交网络——到处都是 Embedding。

可以说，Embedding 是线性代数送给机器学习的一份大礼：**把离散的、无结构的符号世界，映射到连续的、有几何结构的向量空间里。** 一旦进入了向量空间，线性代数的全部工具——内积、投影、线性变换、矩阵分解——就都可以派上用场了。

## Attention 机制：矩阵乘法的华丽变奏

如果说 Embedding 是"把数据塞进向量空间"的入口，那么 **Attention 机制**就是向量进入之后要经历的核心处理。

2017 年，Ashish Vaswani 等人在论文"Attention Is All You Need"中提出了 **Transformer** 架构——一个几乎完全由 Attention 机制搭建的模型。后来的 GPT、BERT、以及今天所有的大语言模型，都是 Transformer 的后代。

Attention 的核心公式是：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

乍一看公式密密麻麻，但如果你有了前面章节的基础，其实可以一层一层拆开看。

**$Q$、$K$、$V$ 从哪来？** 都是通过线性变换得到的。假设输入是一组向量（比如一句话中每个词的 Embedding），堆成一个矩阵 $X$：

$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$

$W_Q$、$W_K$、$W_V$ 是三个不同的权重矩阵（需要训练的参数）。它们分别把每个输入向量"投影"到三个不同的子空间里——Query（查询）空间、Key（键）空间、Value（值）空间。

**$QK^T$ 在做什么？** 这是一个矩阵乘法。如果 $Q$ 有 $n$ 行（$n$ 个词），$K$ 也有 $n$ 行，那 $QK^T$ 就是一个 $n \times n$ 的矩阵——第 $(i, j)$ 个元素是 Query 空间中第 $i$ 个向量和 Key 空间中第 $j$ 个向量的内积。

内积——第十章的老朋友。两个向量的内积衡量它们的"相关程度"。$QK^T$ 的第 $(i, j)$ 个元素，衡量的就是"第 $i$ 个词有多关注第 $j$ 个词"。

**$\sqrt{d}$ 是什么？** 缩放因子。在高维空间里，随机向量的内积的绝对值倾向于随维度 $d$ 增大而增大。除以 $\sqrt{d}$ 是为了把内积的量级稳定在一个合理范围——如果不做这个缩放，内积可能太大，经过 softmax 之后会变成"几乎全是 0 和 1"的极端分布，梯度信号也会因此消失。

**softmax 做什么？** 把每一行变成一个概率分布——所有元素非负，加起来等于 1。这是一个非线性操作。经过 softmax 之后，$n \times n$ 矩阵的每一行变成了一组"注意力权重"——第 $i$ 行告诉你，第 $i$ 个词把多少注意力分配给了其他各个词。

**最后乘以 $V$？** 用注意力权重对 Value 向量做加权平均。每个词的输出是"它所关注的那些词的 Value 向量的加权组合"——关注度高的词贡献大，关注度低的词贡献小。

把整个过程串起来：线性变换（生成 Q、K、V）→ 矩阵乘法（算成对相关性）→ 缩放 → softmax（非线性归一化）→ 矩阵乘法（加权汇总）。

Attention 是一个**高度线性代数化的结构**——它的骨架是矩阵乘法和线性变换。但它的表达力不仅来自这些线性操作，还来自 softmax 引入的非线性归一化、多头机制（同时运行多组独立的 QKV 变换，然后拼接结果）、以及 Transformer 中 Attention 层之间穿插的前馈网络（又是矩阵乘法+非线性激活）。

这跟上一章的教训一脉相承：**线性运算提供结构骨架，非线性运算赋予表达能力。** 两者缺一不可。

一个有趣的细节是 $QK^T$ 这个运算。它计算的是所有词对之间的内积——如果句子有 $n$ 个词，就产生 $n^2$ 个内积。这就是为什么 Transformer 的计算复杂度随序列长度二次增长——也是为什么处理超长文本（几万甚至几十万个token）需要各种巧妙的近似方法。$n^2$ 个内积——矩阵乘法的代价，在这里变成了实际的计算瓶颈。

## 高维空间：反直觉的几何

Embedding 向量通常是几百甚至几千维的。大语言模型中的隐藏状态维度动辄 4096、8192 甚至更高。

在这样的高维空间里，我们从三维世界锻炼出来的几何直觉，会严重失灵。

### 随机向量几乎正交

高维空间最重要的反直觉性质之一：**随机独立生成的高维向量，通常近似正交。**

"近似正交"是什么意思？两个向量的夹角接近 90 度——它们的内积接近零（相对于它们的长度而言）。

这里"随机独立生成"是关键的限定条件——不是说高维空间里的"所有"向量都互相正交（那显然是荒谬的），而是说如果你在高维空间里随机均匀地采样向量，它们之间的夹角会高度集中在 90 度附近。

直觉上可以这样理解：在 $d$ 维空间里，两个随机向量的余弦相似度（第十章的内积除以长度之积）的期望值是 0，方差大约是 $1/d$。当 $d$ 很大时——比如 $d = 1000$——余弦相似度的标准差只有约 $1/\sqrt{1000} \approx 0.032$。也就是说，两个随机向量的余弦值几乎一定落在 $[-0.1, 0.1]$ 的范围内——对应的角度在 84 度到 96 度之间。

这意味着什么？意味着高维空间里有**大量几乎独立的方向**。在三维空间里，你最多找到三个互相正交的方向。但在 1000 维空间里，你可以找到远不止 1000 个"几乎正交"的方向——事实上，在余弦相似度阈值设得不太严格的情况下，你可以在 $d$ 维空间里塞进指数级数量的"近似正交"向量。

这就为 Embedding 的可行性提供了数学基础。一个 300 维的词向量空间，理论上有足够的"空间容量"来容纳几万甚至几十万个词的向量，使得不同词之间有足够的几何距离来区分——同时又允许语义相近的词靠得比较近。

### 但训练出来的向量不是随机的

然而，有一个重要的区别需要强调：**训练出来的 Embedding 向量通常不是随机向量。**

随机向量均匀地散布在高维球面上，彼此近似正交。但经过数据训练之后，Embedding 向量的分布往往呈现出显著的**各向异性**（anisotropy）——向量不是均匀散布的，而是倾向于聚集在某些方向上，留下大片"空旷"的区域。

研究者发现，许多预训练语言模型的词向量存在明显的偏置：所有向量的均值不在原点，而是偏向某个方向；向量的分布在少数几个主成分方向上拉得很长，在其他方向上被压得很扁。这就像一个高维的"扁椭球"——大部分向量挤在椭球的长轴附近。

这种各向异性带来的实际后果是：如果你直接用余弦相似度来衡量两个词向量的相似性，结果可能被全局偏置污染——两个词可能仅仅因为它们都靠近那个"偏置方向"而显得相似，而不是因为它们在语义上真的相关。后续研究提出了各种修正方案，比如减去全局均值、去除前几个主成分的投影（又是第十章正交投影和第十五章 PCA 的精神！）。

所以，高维空间"提供了大量近似独立方向"这个数学事实是 Embedding 的基础设施，但训练出来的 Embedding 空间的实际几何形态——聚簇、偏置、各向异性——比理论上的"随机向量均匀分布"要复杂得多。

## 余弦相似度：并非万能的尺子

说到衡量向量之间的关系，最常被提起的工具是**余弦相似度**：

$$\cos(\theta) = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \|\mathbf{b}\|}$$

就是第十章讲的内积除以两个向量的长度之积。它衡量的是两个向量方向上的一致程度，忽略长度差异。值为 1 表示方向完全相同，值为 0 表示正交，值为 -1 表示方向完全相反。

余弦相似度在语义检索中被广泛使用——你输入一个问题，系统把问题编码成向量，然后在数据库中找出余弦相似度最高的文档向量。这是当下"检索增强生成"（RAG）架构的基础操作之一。

但余弦相似度**不是永远优于欧氏距离**。选择哪种度量取决于具体场景。

如果 Embedding 向量已经被 L2 归一化（投影到单位超球面上），那么余弦相似度和欧氏距离是单调关系——$\|\mathbf{a} - \mathbf{b}\|^2 = 2(1 - \cos\theta)$——选哪个都一样。

但如果向量没有归一化，两者就会给出不同的排序。余弦相似度只看方向，不看"强度"；欧氏距离同时考虑方向和长度。在某些任务中——比如衡量用户兴趣的"强度"而不仅仅是"类型"——长度信息可能是有用的，这时候欧氏距离或者内积可能比余弦相似度更合适。

说到底，余弦相似度的流行不是因为它有数学上的绝对优越性，而是因为它在很多经验场景中表现不错，而且它有一个很直观的解释——"方向一致性"。但在严肃的工程实践中，度量的选择应该基于具体任务的验证，而不是教条。

## 从高维几何到大模型的"理解"

让我们把这一章的线索串起来。

**Embedding** 把离散符号映射为高维向量，在向量空间中建立起语义的几何结构。**Attention** 通过矩阵乘法计算向量之间的成对相关性，让模型能够动态地"关注"输入中最相关的部分。**高维空间** 提供了大量近似独立的方向，为丰富的语义表示提供了数学上的"容量保证"。

这三者合在一起，构成了大语言模型的数学骨架。

当你问 ChatGPT 一个问题时，你的文字先被 Embedding 变成一组高维向量，然后这些向量经过几十层 Transformer——每一层都是矩阵乘法、softmax、矩阵乘法、更多矩阵乘法——最终输出一个向量，这个向量通过又一次矩阵乘法映射回词汇表空间，softmax 把它变成概率分布，然后从中采样出下一个词。

从头到尾，**每一步都在做向量空间里的运算**。线性变换把向量从一个表示空间映射到另一个。内积衡量向量之间的相关性。矩阵乘法汇聚来自不同位置的信息。非线性激活在矩阵运算之间注入表达力。

这台机器"理解"语言吗？关于这个哲学问题，本书不做判断。但从数学的角度，我们可以说一件更朴素的事情：**这台机器展示了，在足够高维的向量空间里，通过足够多层的矩阵运算和非线性变换，可以捕捉到人类语言中惊人丰富的统计规律和结构模式。**

"捕捉统计规律"听起来不如"理解语言"那么浪漫。但当你想到这些"统计规律"来自几百维向量之间的内积和线性变换——来自凯莱在 1858 年定义的矩阵乘法、来自格拉斯曼在 1844 年构想的高维向量空间、来自欧几里得两千多年前就在研究的长度与角度（它们如今被统一在内积之中）——你或许会对这些古老的数学概念生出一种新的敬畏。

## 线性代数遇见语言

让我们退后一步，看看这一章真正在讲什么。

从 Word2Vec 的惊人等式，到 Embedding 层的"查表即线性变换"，到 Attention 机制中的三重矩阵乘法，到高维空间里"随机向量近似正交"的反直觉性质——大模型的每一个核心组件，几乎都能在前面章节的工具箱里找到对应的概念。

Embedding 是第七章线性变换的实例。Attention 中的 $QK^T$ 是第十章内积的大规模版本。余弦相似度是第十章正交性理论的直接应用。高维空间里的近似正交性，依赖的是第六章定义的维数概念。

但这里有一个值得玩味的地方：这些工具在十九世纪和二十世纪初被发明的时候，没有人想到它们会被用来处理语言。格拉斯曼定义高维向量空间的时候，想的是几何；凯莱研究矩阵乘法的时候，想的是方程组。一百五十年后，这些抽象工具竟然成了让机器"理解"人类语言的基础设施。

这不是巧合，而是抽象的力量——当你把概念从具体场景中剥离出来，它就获得了在完全不同的场景中复用的可能性。

走到这里，第三幕的五章故事讲完了。从计算革命到矩阵分解，从 PageRank 到推荐系统，从神经网络到大模型——线性代数在每一个新领域都不只是"被应用"，而是成为了思考问题的基本语言。

而现在，是时候停下来，回头看看我们走过的整条路了。
