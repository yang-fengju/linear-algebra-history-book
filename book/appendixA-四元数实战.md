# 附录A：四元数——从"看不懂"到"会用"

> 第五章讲了四元数是怎么被发现的。这篇附录讲一件更实际的事：它到底怎么用。

---

第五章里，我们跟着哈密顿走过了四元数的诞生过程——十年的死胡同、布鲁姆桥上的灵光、$i^2 = j^2 = k^2 = ijk = -1$。你现在知道四元数是什么、为什么是四维、乘法表怎么来的。

但你可能还是觉得：我能复述这个故事，却不知道四元数到底怎么"用"。$a + bi + cj + dk$ 四个数，每个到底代表什么？怎么用它来旋转一个三维物体？凭什么说它比别的方法好？

这篇附录就来回答这些问题。不讲历史了，直接拆机器。

---

## 先拆开看看：四个数各是什么意思

一个四元数长这样：

$$q = a + bi + cj + dk$$

四个实数。哈密顿把它分成两部分：**标量部分** $a$（一个数）和**向量部分** $bi + cj + dk$（三个数，对应三维空间的三个方向）。

但这么说还是太抽象。让我换一个说法。

把四元数想象成一张**旋转指令卡**。卡上写了两条信息：

1. **绕什么轴转？**——向量部分告诉你旋转轴的方向
2. **转多少度？**——标量部分告诉你旋转的角度

具体来说，如果你想绕一个轴旋转角度 $\theta$，旋转轴的方向是单位向量 $\hat{u} = (u_x, u_y, u_z)$，那对应的四元数是：

$$q = \cos\frac{\theta}{2} + \sin\frac{\theta}{2}(u_x i + u_y j + u_z k)$$

这就是"旋转四元数"的标准公式。注意两个要点：

**要点一：角度除以了 2。** 标量部分不是 $\cos\theta$，而是 $\cos(\theta/2)$。向量部分的长度也不是 $\sin\theta$，而是 $\sin(\theta/2)$。为什么是半角？因为四元数旋转用的是"三明治"操作（下一节会讲），左右各作用一次，每次贡献半个角度，合起来刚好是 $\theta$。就像你用两只手各转半圈来翻一张扑克牌——每只手只负责一半。

**要点二：模长为 1。** $\cos^2(\theta/2) + \sin^2(\theta/2) = 1$，所以旋转四元数的模长恒为 1。这跟复数里 $\cos\theta + i\sin\theta$ 住在单位圆上是一回事——单位四元数住在四维空间的"单位球面"上。

---

来看几个具体的例子，帮你把公式和直觉对上。

**不旋转**（$\theta = 0$）：

$$q = \cos 0 + \sin 0 \cdot (\ldots) = 1$$

四元数就是 $1$。标量部分等于 1，向量部分全是 0。不旋转就是"什么也不做"。

**绕 $z$ 轴旋转 90°**（$\theta = 90°$，轴 $= (0, 0, 1)$）：

$$q = \cos 45° + \sin 45° \cdot k = \frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} k$$

标量部分是 $\cos 45° \approx 0.707$，$k$ 的系数也是 $0.707$，$i$ 和 $j$ 的系数是 0。旋转轴朝 $z$ 方向，角度是 90°。

**绕 $z$ 轴旋转 180°**（$\theta = 180°$，轴 $= (0, 0, 1)$）：

$$q = \cos 90° + \sin 90° \cdot k = 0 + 1 \cdot k = k$$

标量部分变成了 0，四元数就等于 $k$。整个旋转信息全部编码在了向量部分里。

**绕 $x$ 轴旋转 90°**（$\theta = 90°$，轴 $= (1, 0, 0)$）：

$$q = \cos 45° + \sin 45° \cdot i = \frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} i$$

跟绕 $z$ 轴那个长得很像，只不过虚数单位从 $k$ 变成了 $i$——旋转轴从 $z$ 方向变成了 $x$ 方向。

看出规律了吗？$i$、$j$、$k$ 分别对应三个坐标轴方向。向量部分 $(b, c, d)$ 就是旋转轴在三个方向上的分量，乘以 $\sin(\theta/2)$。标量部分 $a = \cos(\theta/2)$ 编码角度。四个数，压缩了"轴 + 角"的全部信息。

## 旋转的"三明治"：四元数怎么转一个点

好，你现在知道怎么构造一个旋转四元数了。下一个问题：怎么用它来真正旋转一个三维空间中的点？

假设你有一个点 $P = (x, y, z)$，想绕某个轴旋转某个角度，求旋转后的新坐标。

分三步走。

**第一步：把点变成四元数。**

把点 $P = (x, y, z)$ 写成一个"纯四元数"——标量部分为 0，向量部分就是点的坐标：

$$p = 0 + xi + yj + zk$$

为什么要这么做？因为点要参与四元数乘法，所以必须先"说四元数的语言"。标量部分设为 0，表示这不是一个旋转指令，而是一个被旋转的对象。

**第二步：构造旋转四元数和它的共轭。**

旋转四元数 $q$ 按前面的公式构造。然后求它的**共轭** $\bar{q}$：

$$\bar{q} = a - bi - cj - dk$$

就是把向量部分全部取反。直觉：如果 $q$ 是"绕某轴正转半个角度"，那 $\bar{q}$ 就是"绕同一个轴反转半个角度"。对单位四元数（模长为 1）来说，共轭就等于逆：$\bar{q} = q^{-1}$。

**第三步：做"三明治"。**

计算：

$$p' = q \cdot p \cdot \bar{q}$$

结果 $p'$ 是一个新的纯四元数（标量部分为 0），它的向量部分 $(x', y', z')$ 就是旋转后的坐标。

为什么要左右各乘一次？为什么不能只乘一次 $qp$？

两个原因。第一，如果只算 $qp$，旋转角度只有 $\theta/2$，不是我们要的 $\theta$。左边的 $q$ 贡献半个角度，右边的 $\bar{q}$ 再贡献半个角度，合起来刚好。

第二，更重要的：只算 $qp$，结果的标量部分通常不为 0——它不再对应一个三维空间中的点。而 $qp\bar{q}$ 的标量部分保证为 0，结果一定是一个合法的三维坐标。三明治操作不仅给了正确的角度，还保证了结果留在三维空间里。

## 一个完整的例子：从头算到尾

光说不练不行。让我们选一个简单的例子，把每一步乘法都算出来。

**问题**：把点 $P = (1, 0, 0)$（$x$ 轴上的单位点）绕 $z$ 轴逆时针旋转 90°。

**预期结果**：应该到 $(0, 1, 0)$（$y$ 轴方向）。你可以先在脑子里想象一下——$x$ 轴正方向的点，绕 $z$ 轴转 90°，确实会到 $y$ 轴正方向。

现在开始算。

**第一步**：把点写成纯四元数。

$$p = 0 + 1 \cdot i + 0 \cdot j + 0 \cdot k = i$$

**第二步**：构造旋转四元数。旋转轴 $\hat{u} = (0, 0, 1)$（$z$ 轴），旋转角 $\theta = 90°$。

$$q = \cos 45° + \sin 45° \cdot k = \frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} k$$

共轭：

$$\bar{q} = \frac{\sqrt{2}}{2} - \frac{\sqrt{2}}{2} k$$

为了书写简洁，设 $c = \frac{\sqrt{2}}{2} \approx 0.707$。那么 $q = c + ck$，$\bar{q} = c - ck$。

**第三步**：计算 $qp$。

$$qp = (c + ck) \cdot i$$

展开：

$$= c \cdot i + ck \cdot i$$

第一项：$c \cdot i = ci$。

第二项：$ck \cdot i = c(ki)$。查乘法表，$ki = j$。所以 $c(ki) = cj$。

合起来：

$$qp = ci + cj$$

**第四步**：计算 $(qp)\bar{q}$。

$$(ci + cj)(c - ck)$$

展开成四项：

$$= ci \cdot c + ci \cdot (-ck) + cj \cdot c + cj \cdot (-ck)$$

$$= c^2 i - c^2(ik) + c^2 j - c^2(jk)$$

逐一查乘法表：

- $ik = -j$，所以 $-c^2(ik) = -c^2(-j) = c^2 j$
- $jk = i$，所以 $-c^2(jk) = -c^2 i$

代入：

$$= c^2 i + c^2 j + c^2 j - c^2 i$$

$$= 2c^2 j$$

而 $c^2 = (\frac{\sqrt{2}}{2})^2 = \frac{1}{2}$，所以 $2c^2 = 1$。

$$p' = j$$

翻译回三维坐标：$p' = 0 + 0i + 1j + 0k$，对应点 $(0, 1, 0)$。

**结果正确！** $(1, 0, 0)$ 绕 $z$ 轴转 90°，确实到了 $(0, 1, 0)$。

---

再来一个稍有难度的，展示四元数的真正威力。

**问题**：把点 $P = (1, 0, 0)$ 绕轴 $\hat{u} = (0, 0, 1)$（还是 $z$ 轴）旋转 120°。

**预期结果**：$x$ 轴上的点绕 $z$ 轴转 120°，应该到 $(-\frac{1}{2}, \frac{\sqrt{3}}{2}, 0)$。

构造旋转四元数：$\theta = 120°$，半角 $= 60°$。

$$q = \cos 60° + \sin 60° \cdot k = \frac{1}{2} + \frac{\sqrt{3}}{2} k$$

$$\bar{q} = \frac{1}{2} - \frac{\sqrt{3}}{2} k$$

计算 $qp$（$p = i$）：

$$qp = (\frac{1}{2} + \frac{\sqrt{3}}{2} k) \cdot i = \frac{1}{2} i + \frac{\sqrt{3}}{2}(ki) = \frac{1}{2} i + \frac{\sqrt{3}}{2} j$$

计算 $(qp)\bar{q}$：

$$(\frac{1}{2} i + \frac{\sqrt{3}}{2} j)(\frac{1}{2} - \frac{\sqrt{3}}{2} k)$$

展开：

$$= \frac{1}{4} i - \frac{\sqrt{3}}{4}(ik) + \frac{\sqrt{3}}{4} j - \frac{3}{4}(jk)$$

查乘法表：$ik = -j$，$jk = i$。

$$= \frac{1}{4} i + \frac{\sqrt{3}}{4} j + \frac{\sqrt{3}}{4} j - \frac{3}{4} i$$

$$= -\frac{1}{2} i + \frac{\sqrt{3}}{2} j$$

对应点 $(-\frac{1}{2}, \frac{\sqrt{3}}{2}, 0)$。跟预期完全一致。

这两个例子都是绕坐标轴转的，你可能觉得没什么了不起。但四元数的真正威力在于：绕任意轴旋转也是同样的三步——只是 $q$ 的向量部分不再只有一个非零分量而已。旋转矩阵做"歪轴"旋转需要先分解成三次坐标轴旋转再合成，四元数一步到位。

## 两次旋转怎么合成？乘一下就行

现实中你经常需要连续做好几次旋转。比如飞机先偏航（绕竖直轴转），再俯仰（绕侧面轴转），再滚转（绕前进方向轴转）。三次旋转的合成效果是什么？

如果用旋转矩阵，你需要乘三个 $3 \times 3$ 矩阵——每次乘法涉及 27 次数字相乘和 18 次相加。

如果用四元数，只需要把三个四元数乘起来。四元数乘法每次只涉及 16 次数字相乘和 12 次相加，比矩阵乘法省不少。

更重要的是概念上的简洁：**两次旋转的合成 = 两个四元数相乘。** 就这么简单。

来看一个具体的例子。

**问题**：先绕 $z$ 轴转 90°，再绕 $x$ 轴转 90°。合成的效果等价于绕哪个轴转多少度？

第一次旋转：$q_1 = \cos 45° + \sin 45° \cdot k = \frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} k$

第二次旋转：$q_2 = \cos 45° + \sin 45° \cdot i = \frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} i$

合成旋转 = $q_2 q_1$（注意：先做的放右边！就像函数复合 $f(g(x))$ 里 $g$ 在内层一样）。

$$q_2 q_1 = (\frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} i)(\frac{\sqrt{2}}{2} + \frac{\sqrt{2}}{2} k)$$

展开：

$$= \frac{1}{2} + \frac{1}{2} k + \frac{1}{2} i + \frac{1}{2}(ik)$$

查乘法表：$ik = -j$。

$$= \frac{1}{2} + \frac{1}{2} i - \frac{1}{2} j + \frac{1}{2} k$$

标量部分 $a = \frac{1}{2}$，向量部分 $(b, c, d) = (\frac{1}{2}, -\frac{1}{2}, \frac{1}{2})$。

从这个四元数反读出旋转信息：$a = \cos(\theta/2) = \frac{1}{2}$，所以 $\theta/2 = 60°$，$\theta = 120°$。旋转轴的方向正比于向量部分 $(1, -1, 1)$，归一化后是 $\frac{1}{\sqrt{3}}(1, -1, 1)$。

**结论**：先绕 $z$ 轴转 90° 再绕 $x$ 轴转 90°，等价于绕 $(1, -1, 1)$ 方向的轴旋转 120°。

如果反过来——先绕 $x$ 轴转 90°，再绕 $z$ 轴转 90°——你可以自己算 $q_1 q_2$，会发现结果不一样。向量部分变成了 $(1, 1, -1)$ 方向。这就是第五章说的"三维旋转不交换"——先转头再低头，和先低头再转头，最终姿态不同。四元数乘法忠实地反映了这一点。

## 四元数赢在哪里

描述三维旋转有三种主流方法：欧拉角、旋转矩阵、四元数。它们各有优劣，但四元数在大多数实际场景中是最优选择。

### 欧拉角：直觉好但有致命伤

欧拉角用三个角度来描述旋转——比如"先绕 $x$ 轴转 $\alpha$，再绕 $y$ 轴转 $\beta$，最后绕 $z$ 轴转 $\gamma$"。只用三个数，简单直观，这是它的优点。

但它有一个致命缺陷：**万向节死锁**。

想象你站直了，规定三步旋转：先左右摇头、再前后点头、最后左右歪头。现在第二步让你"低头 90°"——你的脸朝下了。这时候，第一步的"左右摇头"和第三步的"左右歪头"变成了完全同一个动作——都是绕你身体的竖直轴旋转。两个原本不同的自由度"重叠"了，你丢掉了一个方向的控制能力。有些姿态你到不了了。

这不是理论上的稀奇事。阿波罗 11 号登月任务的惯性导航平台就用了三个物理万向节，宇航员必须时刻注意避开死锁姿态——如果飞船的姿态接近死锁位置，他们需要手动调整来"躲开"。

### 旋转矩阵：准确但笨重

旋转矩阵是一个 $3 \times 3$ 的数表，9 个数。但三维旋转只有 3 个自由度（绕哪个轴、转多少度），所以 9 个数里有 6 个是冗余的，被各种约束条件锁死。

问题在于：连续做很多次旋转之后，浮点运算的误差会慢慢积累，让矩阵不再严格"正交"——行列式不再精确等于 1，行向量不再精确互相垂直。这时候你需要做一次 Gram-Schmidt 正交化来修正，这本身又是一堆计算。

四元数遇到同样的漂移问题怎么办？除以自己的模长就行——四个数各除同一个数，一行代码搞定。

### 四元数的独门武器：Slerp

三种方法里只有四元数天然支持一种叫 **Slerp**（球面线性插值）的操作。

想象你在做动画：一个飞机从朝东变成朝北，中间应该怎么过渡？不能"突然扭头"，得"匀速转弯"。Slerp 能在两个姿态之间生成任意多个中间帧，而且每一帧的旋转角速度恒定——就像一个匀速旋转的表针。

欧拉角做插值会走出奇怪的"之"字路径。旋转矩阵做插值更糟——两个正交矩阵的平均值不再是正交矩阵，结果会把物体压扁或拉伸。只有四元数的 Slerp 能保证插值路径始终走"最短弧线"，而且速度均匀。

### 总结对比

| | 欧拉角 | 旋转矩阵 | 四元数 |
|---|---|---|---|
| 用几个数 | 3 | 9 | 4 |
| 万向节死锁 | **有** | 无 | 无 |
| 合成旋转 | 很麻烦 | 矩阵乘法 | 四元数乘法（更快） |
| 平滑插值 | 很差 | 很差 | **天然支持 Slerp** |
| 修正数值漂移 | 不适用 | 重新正交化（费力） | 归一化（除一下模长） |
| 直觉 | 好 | 一般 | 需要学习 |

四元数唯一的缺点就是不太直觉——这也是你现在正在读这篇附录的原因。但一旦理解了"标量部分 = 半角余弦、向量部分 = 轴方向 × 半角正弦、旋转 = 三明治"这三条，它就跟骑自行车一样，学会了就不会忘。

## 四元数今天在哪里干活

你可能觉得四元数离日常生活很远。其实它就在你手边。

**游戏引擎**。Unity 和 Unreal Engine 里的每一个游戏物体——角色、相机、子弹、车辆——它们的朝向都是用四元数存储的。Unity 的 `Quaternion` 类型就是四元数。角色转身、相机跟随、物理碰撞后的旋转、动画中的姿态插值——全是四元数运算。

你可能用过 Unity 里的 `Quaternion.Euler(x, y, z)` 来输入欧拉角，觉得"我用的不是四元数啊"。实际上，这个函数做的第一件事就是把欧拉角转换成四元数，所有计算都在四元数空间里完成，最后再把结果转回欧拉角显示给你。欧拉角只是给人看的界面，四元数才是引擎的内部语言。

**手机里的陀螺仪**。你手机里有加速度计和陀螺仪，每秒几百次报告手机的旋转角速度。把这些微小的旋转累积起来——也就是不断地把新的旋转四元数乘到当前姿态上——就得到了手机此刻的朝向。手机屏幕的自动旋转靠它，AR 应用把虚拟物体叠在真实桌面上靠它，VR 头盔追踪你的头部运动也靠它。如果用旋转矩阵累积，误差会很快让矩阵"变形"；如果用欧拉角累积，随时可能撞上万向节死锁。四元数没这些问题。

**航天器姿态控制**。卫星、空间站、火箭——任何需要在太空中精确控制自身朝向的东西，现在普遍用四元数来描述和计算姿态。国际空间站的姿态控制系统就是基于四元数的。SpaceX 的猎鹰火箭在着陆时需要实时调整姿态，底层也是四元数在干活。

**机器人和自动驾驶**。机械臂的关节旋转、无人机的姿态估计、自动驾驶中激光雷达点云的坐标配准——这些领域的标准工具链（比如 ROS，机器人操作系统）里处处是四元数。

哈密顿 1843 年在布鲁姆桥上刻下的那个公式，沉寂了一百多年，因为计算机时代的到来而复活。今天，全世界每一部手机、每一个 3D 游戏、每一颗卫星里都在用它。那十年的苦熬，值了。

## 速查卡片

最后，把这篇附录里的关键公式集中放在一起，方便你随时回来查。

**旋转四元数的构造**（绕单位轴 $\hat{u}$ 旋转角度 $\theta$）：

$$q = \cos\frac{\theta}{2} + \sin\frac{\theta}{2}(u_x i + u_y j + u_z k)$$

**三明治旋转**（旋转点 $P = (x, y, z)$）：

$$p = xi + yj + zk \quad \longrightarrow \quad p' = q \, p \, \bar{q}$$

**共轭**（向量部分取反，对单位四元数等于逆）：

$$\overline{a + bi + cj + dk} = a - bi - cj - dk$$

**模长**：

$$|q| = \sqrt{a^2 + b^2 + c^2 + d^2}$$

**归一化**（修正数值漂移）：

$$q_{\text{normalized}} = \frac{q}{|q|}$$

**旋转合成**（先做 $q_1$ 再做 $q_2$）：

$$q_{\text{合成}} = q_2 \cdot q_1$$

**四元数乘法**（两个四元数 $q_1 = a_1 + b_1 i + c_1 j + d_1 k$ 和 $q_2 = a_2 + b_2 i + c_2 j + d_2 k$ 的乘积）：

$$q_1 q_2 = \begin{cases} a = a_1 a_2 - b_1 b_2 - c_1 c_2 - d_1 d_2 \\ b = a_1 b_2 + b_1 a_2 + c_1 d_2 - d_1 c_2 \\ c = a_1 c_2 - b_1 d_2 + c_1 a_2 + d_1 b_2 \\ d = a_1 d_2 + b_1 c_2 - c_1 b_2 + d_1 a_2 \end{cases}$$

**基本乘法表**（第五章推导过的）：

$$ij = k, \quad jk = i, \quad ki = j$$
$$ji = -k, \quad kj = -i, \quad ik = -j$$
$$i^2 = j^2 = k^2 = -1$$

**从四元数反读旋转信息**：

$$\theta = 2 \arccos(a), \quad \hat{u} = \frac{(b, c, d)}{|(b, c, d)|}$$
