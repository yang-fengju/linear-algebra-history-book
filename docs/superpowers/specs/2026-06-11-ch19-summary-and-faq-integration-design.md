## 一、ch13 增强设计

### 改动目标

修正概念全景图段落，解决读者困惑："向量空间是谁？矩阵A是谁？它俩什么关系？"

### 改动内容

在现有概念全景图之前，插入一段"线性代数到底在研究谁"的简明回答：

**核心表述：**

```
线性代数研究的不是某一个孤立对象，而是一组关系：

输入空间 V ── 线性变换 T ──> 输出空间 W
输入向量 x ──  矩阵 A 操作 ──> 输出向量 b

向量空间是向量住的地方。
线性变换是空间之间保持线性结构的动作。
矩阵 A 是这个动作在选定坐标系后写出来的记录——
更精确地说，是给输入空间和输出空间各选定一组基之后的坐标表示。
没有选基，线性变换本身不是一张数字表；选了基，它才被记录成矩阵。

Ax=b 只是这条主轴在坐标里的最常见写法。
```

**修正概念树结构（从单根树改为双主角并列）：**

```
线性代数的两个主角

1. 向量空间（舞台）：向量住在哪里
   ├── 基、维数、子空间
   └── + 内积 → 有了长度、角度、距离、正交

2. 线性变换（动作）：向量怎样被送到另一个空间
   ├── 矩阵表示（选基后的坐标记录）
   ├── 核、像、秩（保留和丢失了什么）
   ├── 行列式（体积缩放多少，方阵）
   └── 特征值、特征向量（哪些方向不变）
         ├── 对角化
         └── 谱定理

3. 两者交汇
   ├── 正交投影、最小二乘（舞台的度量 + 可达输出的最近近似）
   ├── 二次型（变换 A 与内积结构共同产生的能量/曲率）
   └── SVD（任意变换在两组正交坐标系下的标准形：
         输入空间换正交坐标 → 沿主方向缩放 → 输出空间换正交坐标）
```

注：迹（trace）是方阵的另一个经典标量不变量（= 特征值之和），在ch11正文中已有介绍，概念树中不单独列为节点，避免增加读者负担。

**过渡句（替换原有表述）：**

> 这张图解释的是理论骨架。第三幕不会推翻它，只会展示它怎样变成计算基础设施——SVD变成数据压缩算法，特征向量变成PageRank，矩阵乘法变成GPU的日常工作。等你走完第三幕，我们会在新的一章里画出包含一切的完整地图。

### 不做的事

- 不展开五种视角
- 不讲七个核心式子
- 不深入"研究x还是研究A"
- 以上全部留给新ch19

### 章节结构

#### 第一节：先把答案说清楚

开场直接回答全书追问的核心问题：

```
线性代数研究的主轴：

输入空间 V ── 线性变换 A/T ──> 输出空间 W
输入向量 x ──   矩阵 A 操作  ──> 输出向量 b
```

- 向量空间是舞台，线性变换是在舞台上发生的动作，矩阵是动作在某组基下的坐标记录
- Ax=b 只是这条主轴在坐标里的最常见写法
- 回顾ch13的双主角结构，但这次不讲历史，讲地图

**不展开太多**——先钉住答案，后面逐节展开。

#### 第二节：一个式子，五种读法

以 Ax=b 为锚点，用一个具体的 2×2 算例贯穿，展示同一个式子的五种基础视角：

| 视角 | 怎么读 Ax=b |
|------|------------|
| 方程视角 | 几个未知数、几个方程，求解 |
| 行视角 | 每一行是一个约束（一条线/一个平面），解是交点 |
| 列视角 | b 能不能由 A 的列向量线性组合出来 |
| 变换视角 | 向量 x 经过 A 变换后变成了 b |
| 空间视角 | 列空间、零空间、秩——什么信息被保留，什么丢失了 |

同一个算例展示同一个等式。收束：**五种不是五个理论，是五副眼镜看同一件事。**

在五种基础视角之上，点出更多工具性视角（度量视角、谱视角、能量视角、数值视角、高维几何视角），每个用一句话说明它解决什么新问题、对应哪一章。

FAQ来源：Q25。

#### 第三节：研究重心的三次升级

Q30的核心洞察展开：

```
第一层：给我A和b，求x            ← 研究输入输出关系
第二层：A把空间怎么了             ← 研究变换的行为（核、像、秩）
第三层：A本身长什么样             ← 研究变换的内部结构（特征值、分解）
```

比喻：
- 第一层："用这台机器加工零件"
- 第二层："这台机器能加工什么、会废掉什么"
- 第三层："拆开机器看里面的齿轮"

三层对应的章节和概念：

| 层次 | 核心问题 | 对应章节 | 核心概念 |
|------|---------|---------|---------|
| 第一层 | 求x | ch1-ch3 | 方程组、消元法、行列式（判别） |
| 第二层 | A的行为 | ch7-ch10 | 线性变换、核、像、秩、行列式（体积）、正交投影、最小二乘（到不了b时找最近输出） |
| 第三层 | A的结构 | ch11-ch12, ch14-ch15 | 特征值、对角化、SVD、二次型、数值分解 |

注：ch10的正交、投影、最小二乘属于第二层——它仍然在研究A与b的关系（A到不了b时怎么办），只是引入了内积/度量结构来度量"最近"。ch4-ch6（矩阵、向量、向量空间的建立）是语言和舞台的搭建，不属于任何一层的"问题类型"。

FAQ来源：Q30。

#### 第四节：视角是怎么一步步长出来的

Q26的链条改写为叙事。**每一环只用一两句话**，点出"什么问题逼出了什么视角"：

```
方程（怎么求解）
  → 行列式（对方阵系数系统，能不能不解就判断是否退化、是否有唯一解）
    → 矩阵（系数表能不能整体操作）
      → 向量（方向、大小、组合能不能成为独立对象）
        → 向量空间（向量住在哪里，什么是"空间"）
          → 线性变换（空间之间的关系）
            → 内积（怎么量长度、角度、距离）
              → 特征值（哪些方向在变换下不变）
                → 分解（怎么把变换拆开看内部）
                  → 高维几何（万物皆向量后怎么比较）
```

与ch13的区别：
- ch13讲"公理化运动怎么把这些概念统一成体系"（**历史故事**）
- 本节讲"每个视角因为什么问题而诞生"（**逻辑链条**）
- 一个是"怎么统一的"，一个是"为什么会长出这么多层"

FAQ来源：Q26。

#### 第五节：七个核心式子的家族

线性代数围绕的式子不止 Ax=b 一个。七个核心式子构成一个家族——它们之间有亲缘关系但不是简单的父子传承：

| 式子 | 问的问题 | 对应章节 |
|------|---------|---------|
| Ax = b | 谁变换后等于 b（解方程） | ch1-ch2 |
| AX = B | 同一个 A，批量解多组方程；或变换复合 | ch7 |
| Ax = λx | 谁在变换下方向不变（特征值） | ch11 |
| A = 分解（LU/QR/SVD/PDP⁻¹） | A 的内部结构是什么 | ch14-ch15 |
| min_x ‖Ax - b‖² | 无解时最接近的解是什么（最小二乘） | ch10 |
| AᵀAx = Aᵀb | 正交投影后的精确方程（正规方程） | ch10 |
| xᵀAx | 变换 A 与内积结构共同产生的能量/曲率（二次型） | ch12 |

每个式子配一句"人话翻译"和它解决的问题。

注意：xᵀAx 不是 Ax=b 的分支——它是"输入x被A变换后，再和x自己做内积"的度量结构，同时依赖变换和内积。用"家族"（亲缘关系）而非"家谱"（世系传承）来描述它们的关系。

FAQ来源：Q28, Q29。

### 处理原则

1. **不原样搬运Q&A格式**——提取洞察，转化为正文中的解释、算例、过渡或伏笔
2. **每个FAQ洞察只转化成一个正文功能**——解释一个定义、补一个算例、修一个过渡、或埋一个伏笔，不做mini-essay
3. **三层分工**——章节内解决当下卡点；新ch19重组为全书地图；FAQ文件保留为速查索引

#### ch07 线性变换（中改——三个卡点增强）

**不逐条追加12个FAQ，而是围绕三个核心认知障碍增强正文。**

| 卡点 | 覆盖FAQ | 处理方式 |
|------|---------|---------|
| 卡点一：矩阵乘向量到底在干什么 | Q3-Q7, Q9 | Ax = x₁×第1列 + x₂×第2列 + ...。矩阵的每一列是基向量变换后的去处。m×n矩阵把n维输入变成m维输出。矩阵的列记录了基向量被送到哪里——列向量是记录方式，不是变换身份的来源。按列看=基向量去了哪，按行看=每行是一个探测器 |
| 卡点二：矩阵乘矩阵到底在干什么 | Q10-Q11, Q16-Q17 | AB的三种等价读法：(1)复合变换，(2)A逐列作用在B上，(3)先组装变换、向量还没来。AB≠BA是因为非均匀变换的顺序影响结果，用2×2拉伸+旋转算例 |
| 卡点三：V、W、u、v的人话翻译 | Q12-Q13 | V=输入住的空间，W=输出住的空间，u/v=任意两个输入样本。方阵时V和W是同一个空间。定义写两个字母只是为了覆盖所有情况 |

#### ch08 秩核与像（小改）

| FAQ | 处理方式 |
|-----|---------|
| Q14 | 方阵形式上不改变维度（输入输出都是n维），但可以把向量"压"到更小的子空间——像空间维度=秩，秩<n时信息丢失 |
| Q20 | 解的结构：唯一解是一个向量，无穷多解是特解+零空间方向的组合。用具体算例展示 |

#### ch09 行列式回归（小改）

| FAQ | 处理方式 |
|-----|---------|
| Q18 | 在几何意义段落，明确推广到任意维：n×n行列式 = n维超体积的缩放倍数。用1维→2维→3维→4维的递进建立信心 |
| Q19 | 补充说明：行列式是矩阵本身的属性，不是某次Ax运算的属性。变换单个向量时行列式不参与计算 |
| Q22 | **正式解释**（ch3只埋了伏笔）：展示消元的交叉项 ad-bc 和平行四边形面积公式的交叉项是同一组运算。代数来源（消元）和几何含义（面积）在这里完成连接 |

#### 附录A 四元数实战（小改）

| FAQ | 处理方式 |
|-----|---------|

## 写作纪律

### 基础写作纪律（继承自原设计文档）

以下纪律适用于所有新增和修改内容，完整版本见 `2026-06-09-linear-algebra-history-book-design.md`。

**最高优先级：这是一本风趣的科普读物，不是论文**

- **绝对不能写成论文腔**。调性是"懂行的朋友跟你聊天"，不是学者在写综述
- 每一段都要问自己：一个对线性代数好奇但还没学过的人，读到这里会觉得有趣还是想睡觉？
- 设计文档中的精度约束应当自然融入叙事，而不是以脚注或括号注释的形式出现
- 公式出现时，前后必须有直觉解释和具体例子，不能让公式"裸奔"
- 人物故事、历史场景、具体算例是主要的叙事载体，抽象概念是从这些载体中"长出来"的

**历史准确性原则**

1. **戏剧化但准确**：叙述可以有戏剧性，但历史事实和数学表述不能有硬伤
2. **避免过度现代化**：不要把现代概念回溯投射到古人身上。用"用现代眼光看"而非"古人已经有了"
3. **避免绝对性表述**：用"之一"、"通常被认为"、"经典方法之一"等措辞，给历史争议留空间
4. **区分近似与精确**：涉及现代应用时，说明工具是"核心技术之一"而非唯一方法
5. **数学表述精确**：行列式限定方阵、矩阵-变换对应限定有限维、方程组判据用完整版本、二次型默认对称等
6. **正式写稿前核实史料**：关键历史节点需要交叉验证可靠来源

### 本次特有的补充约束

1. **ch13与新ch19的职责划分**：ch13讲"统一的历史"（公理化运动怎么把概念统一），新ch19讲"统一后的地图"（统一后长什么样、研究谁、概念怎么连接）。两章的概念树/网络图有重叠节点，但视角不同——ch13是故事，ch19是全景
2. **FAQ融入不是搬运**：提取洞察转化为正文功能（解释/算例/过渡/伏笔），不以Q&A形式出现在正文中
3. **每个FAQ洞察只转化成一个正文功能**：防止同一个问题在多个章节重复展开。章节内解决当下卡点，新ch19负责全局串联
4. **ch7三个卡点是结构性增强**：不是在现有正文末尾追加内容，而是在对应位置增强叙述，可能需要调整段落顺序
5. **概念网络图控制复杂度**：核心网不超过15个节点，应用映射用表格而非第二张图
6. **ch20自指更新**：原ch19顺延为ch20后，必须检查并更新所有自指——"过去十八章"→"过去十九章"、章节编号、与前章承接措辞等
7. **概念网络图必须覆盖ch10**：正交、投影、最小二乘是从精确方程到近似世界的关键桥梁，不能从网络图中遗漏
